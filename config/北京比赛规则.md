# 计算规则

每种准确率（%）=（1－ABS（识别数量－鉴定数量）/鉴定数量）×100
当鉴定数量为 0，识别数量≥1 时，识别准确率按 0 计；按公式计算，得出结果为负数时，识别准确率也按 0 计。
总的识别计数准确率取各害虫准确率的算术平均值。

## 生产环境
报出所有害虫，按照上述计算方法，最后每类通过算术平均来计算最后的分数。 
按照配置报出的所有昆虫都需要计算， 背景虫不报出。

## 摆拍环境
保持TOP1/TOP2/TOP3/Other类别， 其中TOP1/TOP2分别是一二类害虫， TOP3是一般害虫。
因为可能识别出标样中实际不存在的虫种或者虫体，所以需要进行修正，公式如下。
修正后的标样识别计数准确率（%）＝MAX〔0，各类标样识别计数准确率的算术平均值－（额外识别出的非标样虫体数量/全部标样虫体实际总数量）×100〕
注意修正的误报出的类别 不做 算术平均计算。

> 外部参考清单：比昂梳理昆虫名单、0625 比赛清单（`doc/参考资料/北京比赛/0625比赛清单.xlsx`）见文末附录。

| 开关 | 含义 |
|:---|:---|
| `report_all_switch=true` | 全部类别报出，不做 tier 过滤 |
| `report_all_switch=false` + `shengchan_switch=true` | **生产模式**：仅报出 `top1_classes` + `top2_classes` |
| `report_all_switch=false` + `shengchan_switch=false` | **摆拍比赛模式**：报出 `top1` + `top2` + `top3`（均为 ``out`` **key**）；`infer_name` 仅对外输出 |

> **key 与 infer_name**：`postprocess` 的 top1/top2/top3/other 清单与 ``out`` 树 **key** 对齐（如 `dongfangnianchong`）；``infer_name``（如 `dongfangzhanchong`）写入结果 ``infer_name`` 字段，供 Pascal VOC 对外输出及外部测试集标签匹配；内部 tier 过滤与 VOC 校验统计按 **key**。

| 分组 | 含义 | 调整策略 |
|:---|:---|:---|
| **top1** | 全国一类害虫 | 基本固定，随国家害虫名录 |
| **top2** | 各省通用二类害虫 | 随国家/多省通用二类名录；含蛴螬类（鳃金龟/丽金龟）、叶蝉类等 |
| **top3** | 一般害虫 | 未列入各省通用二类的农业害虫；含 0625 摆拍目标、重点混淆及算法扩展种；随验证与误报增补 |
| **其他** | 天敌、益虫、中性虫及有样本的低优先级易混淆昆虫；**含大量实际为农业害虫、因样本/报出策略暂留 other 的类群** | 天敌益虫用于抑制误报；植食害虫随验证与名录增补逐步上调 top2/top3 |
| **背景** | `postprocess.background_classes`：非目标虫/干扰（蚊、蝇、隐翅虫、泛尺蛾混淆、水生石蛾等） | 一般不纳入 top 报出 |
