# 关键配置汇总

## run_model（临场切换，改后重启服务）
在 **`config/insect_alg_all.json`** 顶层修改 ``run_model`` 即可切换整套算法配置：

| run_model | 实际加载 |
|:---|:---|
| `baipai` | `insect_alg_all.json`（摆拍） |
| `shengchan` | `insect_alg_shengchan.json`（生产） |
| `other` | `insect_alg_other.json`（其他） |

以下字段**仅在** `insect_alg_all.json` 维护；加载 shengchan/other 时自动从启动配置合并，无需在场景文件中重复配置：

- 顶层 `model_dir`
- 顶层 `predict_cfg`（推理开关，见下）
- `postprocess` 内 `top1_classes` / `top2_classes` / `top3_classes` / `other_classes` / `background_classes`

### predict_cfg（推理配置，仅维护在 insect_alg_all.json）

| 键 | 说明 |
|:---|:---|
| `trt_switch` | TensorRT 总开关，`false` 时全部回退 `.pt` |
| `cls_batch_size` | YOLO 嵌套分类 crop 批量上限（``.pt`` / TRT 共用） |
| `predict_phase_profile` | 分阶段耗时统计 |
| `parallel_detect_seg` | detect / segment 根模型并发 |
| `clip_profiles_enable` | 多尺度 ``clip_profiles`` 滑窗开关 |
| `detect_seg_batch_size` | 滑窗 tile YOLO batch 上限（仅 ``clip_profiles_enable=true``） |

显式传入 ``config/insect_alg_shengchan.json`` 等路径时，不再读取 ``run_model``，直接加载该文件。


# 预处理
## 多尺度检测开关【友商检测开，其他时候关】
``predict_cfg.clip_profiles_enable``（原 ``prepare.clip_profiles_enable``，已迁入推理配置）

## detect_seg_batch_size（默认 32，位于 predict_cfg）
``clip_profiles_enable=true`` 时，detect/seg 滑窗每个 tile 按 batch 调用 YOLO；3060 建议 **8~16**。

## 圆盘处理【友商检测开】
roi_switch 圆盘处理开关

圆盘 ROI 生效后，分割结果若与检测到的圆盘边缘高度重合（贴边弧段误分割），或跨圆盘边界，会被 ``roi_edge`` 过滤。阈值在 ``prepare``：

- ``roi_disk_edge_hug_ratio``（默认 0.55）：分割曲线贴圆盘边缘比例 ≥ 该值 → 过滤
- ``roi_disk_edge_outside_min``（默认 0.02）：贴边且圆外采样点占比 > 该值 → 跨边界过滤

``poly_merge_edge_px`` / ``poly_merge_edge_ratio``（分割根配置）用于计算贴边距离与跨边界判据。

# 后处理

## shenchan_map_classes
生产类名映射表（`key→value`），如 `{"sanhuaming": "erhuaming"}`。配置在对应场景 `postprocess` 中时，在 tier 过滤前将推理类名改写为 value；未命中则保持原类名。

## bin_dark_ratio_min（0.15）
检测框裁切后全图灰度归一化 + Otsu 二值化，较暗簇像素占框内面积须 **≥** 该值，否则过滤（调试图 `bin_dark(0.19<0.20)`）。配置在 `postprocess.bin_dark_ratio_min`；单 detect 根可覆写；`<=0` 关闭。

## debug（postprocess）
`postprocess.debug: true` 时，将稻飞虱 `in_big` 二值化图与分母 mask 拼图保存到 ``{OUTPUT_DIR}/eval_metrics/debug/``。

# 稻飞虱后处理

## skip_dia_conf（0.85）
在稻飞虱嵌套分类节点上配置置信度阈值（如 0.85），当分类 cls_conf 严格大于该值时，跳过本路由及子路由上的 out.*.dia 对角线尺寸过滤，避免高置信度细分类结果被尺寸门限误杀。

## in_big_conf（0.9）
稻飞虱检测框裁切前景 mask 与大虫分割 mask 的 **相交像素 / 大虫 mask 像素**（并集去除小虫独有区域）；相交为 0 不计算（视为正确检出）；比例严格大于该阈值时剔除误报（标签 `i0.85`）。

## in_big_fill（false）
`true` 时 **边缘+二值化种子生长**：Otsu 深色/有色种子 → 种子外接框包络内闭运算与孔洞填充；边框 Lab 剔除托盘空白；禁止全图轮廓泛洪。

## in_big_skip（["wen", "dawen"]）
`in_big_conf` 判定时，忽略清单中的大虫分割类（如蚊、大蚊），不参与 IoU 参考 mask 计算。

# TensorRT 分类批量

## trt_switch（默认 true，位于 predict_cfg）
顶层开关：`false` 时 detect/seg/嵌套 cls **全部**使用 ``model`` 指向的 YOLO ``.pt``，忽略 ``trt`` 字段；改后重启服务。仅维护在 ``insect_alg_all.json`` 的 ``predict_cfg``，shengchan/other 自动继承。

## cls_batch_size（默认 32，推荐 8~16，位于 predict_cfg）
嵌套 `models.cls` 使用 **YOLO11 classify**（``.pt`` 或 TensorRT ``.engine``）时，同一张图多个 crop 拼 batch 推理的上限。配置在 **`insect_alg_all.json` → `predict_cfg`**；单个 ``models.cls`` 节点可覆写。兼容旧键 ``cls_trt_batch_size``。

TRT 时须 **≤ engine 导出时的 max batch**（``acc_tensorRT.py`` 中 ``BATCH``，``DYNAMIC=True``）。3060 12GB + classify 1280 建议 **16**。ConvNeXt/timm 分类仍逐条推理。

## predict_phase_profile（默认 false，位于 predict_cfg）
``pipeline.predict`` 分阶段耗时：``detect``（检测根模型+路由）、``seg``（分割根模型+路由）、``cls-batch``（嵌套 YOLO 分类批量）、``post``（ROI、in_big、report_tier）。跑 ``predict_all.py`` 批量测试时在汇总中打印各阶段占比。

## parallel_detect_seg（默认 true，位于 predict_cfg）
同时存在 detect 与 segment 根模型时，两路在独立线程并发跑图。日志 ``phases[...]`` 中 ``sum`` 为各线程分阶段累加；``wall≈`` 为并发墙钟估计 ``max(detect,seg)+cls+post``。单 GPU 上若 CUDA 串行，墙钟收益可能小于理论值。
